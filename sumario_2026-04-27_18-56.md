## 2. Lo que se corrigió desde el sumario anterior

### ✅ `EtlProcess` — setter roto eliminado, método `change_status` limpio

El setter `status(self, new_status, flag)` que rompía Python fue reemplazado completamente por un método `change_status(flag: bool)`. La lógica es correcta:

```python
def change_status(self, flag: bool) -> None:
    new_status = self._LOOK_UP_TABLE[(self.status, flag)]
    match new_status:
        case PipelineStatus.IDLE:
            self.start_time = None
            self.end_time = None          # ← ya no es doble start_time
        case PipelineStatus.RUNNING:
            self.start_time = datetime.now()
            self.end_time = None
        case PipelineStatus.FAILED | PipelineStatus.SUCCESS:
            self.end_time = datetime.now()
    self.audit.append(
        f"{datetime.now(timezone.utc)} - {self.status} to {new_status}"  # ← correcto
    )
    self._status = new_status
```

Tres bugs corregidos de una vez: setter inválido, audit hardcodeado, doble `start_time`.

```python
def write_lazy(self, data: pl.LazyFrame, **kwargs) -> None:
```

```python
    kwargs.setdefault("table", self.table)
    client.insert_arrow(arrow_table=frame.to_arrow(), **kwargs)
```

## 4. Problemas de diseño que siguen pendientes

### 🟡 `StateMachine` y `EtlProcess` siguen sin conectarse

`EtlProcess` ahora tiene un `change_status` limpio, lo que abre el camino para migrar la `_LOOK_UP_TABLE` inline a `StateMachine[PipelineStatus, bool, EtlProcess]`. Sigue siendo duplicación conceptual.

### 🟡 `ISourcePort` es redundante

`ISourcePort` e `ISourcePortDB` declaran el mismo método `read_lazy` sin relación de herencia entre sí. `ISourcePort` no es usada por ningún adaptador. Debería: `ISourcePortDB(ISourcePort)`.

### 🟡 `ISourcePortFile.file_path` no es abstracta

```python
@property
def file_path(self) -> Path:
    ...   # falta @abstractmethod
```

`CsvSource`, `ExcelSource` y `ParquetSource` asignan `self._file_path` en `__init__` pero nunca implementan la property. El contrato no se enforcea.

### 🟡 `IDestinationPort` usa `Protocol`, los demás `ABC`

`ClickHouseDestination` hereda de `IDestinationPort(Protocol)` mientras que todos los demás puertos usan `ABC`. Inconsistencia sin razón explícita.

### 🟡 Typo persistente: `dependes_on` → `depends_on`

En `EtlData`, `BaseETLUseCase`, y `JSONRegistry`. Mejor corregir antes de que prolifere.

### 🟡 `DbSource.__post_init__` llama `test_connection()` en construcción

Hace imposible instanciar `DbSource` sin base de datos disponible (tests, CI, ambientes sin red). Separar construcción de validación.

### 🟡 `IQueryPort` tiene lógica concreta en `domain/ports`

`get_query_result_to_dataframe` es un método con implementación real en una interfaz de dominio. Es un helper de aplicación; debería vivir en `application/`.

### 🟡 `SqlAlchemyLSourceFactory` — nombre engañoso

No usa SQLAlchemy; usa connectorx/polars.
